### 현재 residual risk / 제한사항
- 현재 레포에는 CRC polynomial/initial/final xor/bit order/counter semantics/window 등 **명시적 메타데이터가 없음**
- `Dbc.fs` 의 `IsCrc` / `IsCounter` 는 signal-name heuristic 이므로, 이것만으로 generated validation semantics를 확정하는 것은 위험
- decode path가 stateless인 현재 구조에서 counter 검증은 추가 상태 모델 또는 caller contract 없이 일반화하기 어려움

### 다음 세션 진입점
1. CRC/Counter metadata schema 설계
   - YAML 기반 explicit signal selection + algorithm parameters
2. 최소 1개 이상의 deterministic fixture 추가
   - happy path / mismatch / disabled / unsupported
3. 그 후 `L-2b`, `L-2c` 구현
