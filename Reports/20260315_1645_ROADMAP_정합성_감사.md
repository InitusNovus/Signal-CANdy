### 고신뢰 발견 사항
1. **[High] L-3a 범위 과장**
   - **ROADMAP claim**: `ROADMAP.md:123` — "시그널 수 > 32인 메시지에서 `uint64_t` 또는 배열 기반 valid 필드 자동 선택"
   - **Observed repo evidence**: `src/Signal.CANdy.Core/Codegen.fs:492`는 33개 초과 mux 메시지에 `uint64_t`만 선택하고, `src/Signal.CANdy.Core/Codegen.fs:1043`는 64개 초과 시 `CodeGenError.UnsupportedFeature`를 반환한다. `tests/oracle/CATEGORY_C_EXCEPTIONS.md:54`도 `>64`는 미지원이라고 정리한다.
   - **Gap/risk**: ROADMAP 문구는 배열 기반 fallback까지 완료된 것으로 읽히지만, 실제 구현은 64비트 확장까지만 존재한다.
   - **Recommendation**: L-3a 문구를 "33-64는 `uint64_t`, >64는 미지원"으로 축소하거나, 배열 기반 fallback을 별도 미완료 backlog로 재분리한다.

2. **[High] L-2 완료와 README 설명 충돌**
   - **ROADMAP claim**: `ROADMAP.md:118`, `ROADMAP.md:119` — CRC/Counter 코드 생성 로직 및 검증 테스트 완료
   - **Observed repo evidence**: `src/Signal.CANdy.Core/Codegen.fs:794`, `src/Signal.CANdy.Core/Codegen.fs:850`에 CRC validate/encode insert 및 counter check 생성이 있고, `tests/Signal.CANdy.Core.Tests/CodegenTests.fs:678`, `tests/Signal.CANdy.Core.Tests/CodegenTests.fs:776`에 해당 테스트가 있다. 반면 `README.md:679`, `README.md:680`, `README.ko.md:148`, `README.ko.md:149`, `README.ko.md:641`, `README.ko.md:642`는 여전히 전체 CRC/Counter 생성 검증이 미구현/추후 YAML 메타데이터 예정이라고 적고 있다.
   - **Gap/risk**: ROADMAP상 완료 기능이 공식 문서에서는 미완료처럼 보인다. 사용자와 유지보수자가 기능 유무를 반대로 이해할 수 있다.
   - **Recommendation**: README EN/KO의 CRC/Counter 설명을 현재 구현 상태(명시적 YAML 메타데이터 + validate/passthrough/fail_fast + CRC-8 only)로 갱신한다.

3. **[High] Tracking source-of-truth가 현재 상태와 어긋남**
   - **ROADMAP claim**: `ROADMAP.md:140`는 Tracking 섹션의 source-of-truth 우선순위에 `tests/oracle/ORACLE_RESULTS.md`를 둔다. `ROADMAP.md:159`, `ROADMAP.md:164`, `ROADMAP.md:169`는 B-O1/B-O2/B-O3를 모두 완료로 체크한다.
   - **Observed repo evidence**: `tests/oracle/ORACLE_RESULTS.md:60`은 여전히 mux skip 33을 남겨두고, `tests/oracle/ORACLE_RESULTS.md:136`, `tests/oracle/ORACLE_RESULTS.md:138`은 raw-range heuristic와 valid auto-widening을 추천 미래 작업으로 적는다. 하지만 `tests/oracle/CATEGORY_C_EXCEPTIONS.md:25`, `tests/oracle/CATEGORY_C_EXCEPTIONS.md:54`, `tests/oracle/CATEGORY_C_EXCEPTIONS.md:91`, `tests/oracle/CATEGORY_C_EXCEPTIONS.md:93`은 해당 항목들을 이미 RESOLVED로 기록한다.
   - **Gap/risk**: Tracking이 의존하는 근거 문서끼리 시점이 어긋나서, 완료 backlog가 미완료처럼 보이거나 corpus 기준선이 잘못 해석될 수 있다.
   - **Recommendation**: `tests/oracle/ORACLE_RESULTS.md`를 현 상태에 맞게 갱신하거나, ROADMAP Tracking의 source-of-truth 우선순위를 실제 최신 문서 순서로 재정렬한다.

4. **[Medium] M-4b의 AGENTS 의존성 정확도 보정이 현재는 다시 깨짐**

   - **Gap/risk**: ROADMAP은 문서 정확도 보정을 완료로 표시하지만, 현재 contributor guidance는 실제 빌드 의존성과 어긋난다.
   - **Recommendation**: M-4b를 현 상태 기준으로 재검증하고 `AGENTS.md` Key Dependencies를 현재 Core 의존성 집합에 맞게 다시 갱신한다.

   - **Gap/risk**: ROADMAP이 직접 가리키는 증빙 일부가 현재 저장소에서 재검증 불가하다. 특히 header-level 근거 문서와 O-1 증빙 체인이 약해진다.
